# v1.0.1

## What's new

- Added a per-profile "fast response" checkbox in the sidebar UI.
- Switching profiles can now optionally write `service_tier = "fast"` to the active provider in `~/.codex/config.toml`.
- The current profile detection and "import current config" flow now include the fast-response state.
- Added clearer UI status for fast response in the profile list and current configuration panel.
